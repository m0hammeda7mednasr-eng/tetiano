# رفع الباك اند على Railway

## المشكلة
Railway شغال على نسخة قديمة من الكود مفيهاش الـ routes الجديدة `/api/app`

## الحل
نعمل redeploy يدوي على Railway عشان يجيب آخر نسخة من GitHub

## الخطوات (أسهل طريقة)

### 1. افتح Railway Dashboard
- روح على: https://railway.app/dashboard
- سجل دخول

### 2. اختار المشروع
- دور على "tetiano" أو اسم الـ backend service
- اضغط عليه

### 3. اعمل Redeploy
- اضغط على تاب **Deployments**
- اضغط على زرار **Deploy** (فوق على اليمين)
- أو اضغط على آخر deployment واختار **Redeploy**

### 4. استنى الـ Build
- هياخد 2-3 دقائق
- اتفرج على الـ logs
- لما تشوف "Build successful" يبقى تمام

### 5. اختبر الـ Deployment
افتح في المتصفح:
```
https://tetiano-production.up.railway.app/health
```

المفروض تشوف:
```json
{"status":"ok","timestamp":"..."}
```

## طريقة تانية: Force Push

لو Railway متوصل بـ GitHub auto-deploy:

```bash
# اعمل commit فاضي عشان يعمل deploy
git commit --allow-empty -m "chore: trigger Railway redeploy"
git push origin main
```

Railway هيلاحظ الـ push ويعمل deploy تلقائي.

## لو في مشكلة

### الـ Deployment فشل

1. **شوف الـ Build Logs**
   - دور على أخطاء TypeScript
   - دور على dependencies ناقصة

2. **تأكد من الـ Environment Variables**
   المتغيرات المطلوبة في Railway:
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_KEY`
   - `FRONTEND_URL`

3. **امسح الـ Cache**
   - في Railway Dashboard
   - Settings → Clear Build Cache
   - اعمل Redeploy تاني

### الـ Routes لسه 404

1. **تأكد من الـ Git Commit**
   ```bash
   git log --oneline -5
   ```
   المفروض تشوف:
   - `71e8cb6` - docs: Add Arabic quick start guide
   - `059a54f` - docs: Add comprehensive fix status report
   - `40543e2` - fix: Add production database schema migration

2. **تأكد من Railway Settings**
   - متوصل بالـ repo الصح
   - بيراقب الـ `main` branch
   - الـ root directory صح

### اختبار محلي

لو عايز تتأكد إن الكود شغال:
```bash
cd backend
npm install
npm run build
npm start
```

لو اشتغل محلي، يبقى المفروض يشتغل على Railway.
