# إصلاح قاعدة البيانات - حل أخطاء 400 و 404

## المشكلة
بعد الـ refactoring الكبير، في مشاكل في production:

### 1. أخطاء 400 (Bad Request)
```
GET user_profiles?select=id,full_name,role,store_id,is_active,avatar_color,permissions
```
الـ columns `store_id` و `permissions` مش موجودة في جدول `user_profiles`

### 2. أخطاء 404 (Not Found)
الـ endpoints الجديدة موجودة في الكود بس الـ database schema قديم

## الحل

### الخطوة 1: تشغيل SQL Script على Supabase

1. افتح Supabase Dashboard: https://supabase.com/dashboard
2. اختار المشروع بتاعك
3. روح على **SQL Editor** من القائمة الجانبية
4. افتح ملف `FIX_PRODUCTION_SCHEMA.sql`
5. انسخ كل المحتوى والصقه في SQL Editor
6. اضغط **Run** أو **Execute**

### الخطوة 2: تأكد من نجاح التنفيذ

بعد تشغيل الـ script، تأكد إن:
- جدول `stores` اتعمل
- جدول `store_memberships` اتعمل
- جدول `shopify_connections` اتعمل
- الـ columns الجديدة اتضافت لـ `user_profiles`

### الخطوة 3: اختبر الموقع

1. افتح الموقع: https://tetiano.vercel.app
2. سجل دخول
3. تأكد إن الأخطاء اختفت من Console
