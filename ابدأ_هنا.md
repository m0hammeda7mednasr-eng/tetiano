# ابدأ هنا - إصلاح الأخطاء في Production

## 🚨 المشكلة الحالية

الموقع فيه أخطاء 400 و 404 بسبب إن الـ database schema قديم

## ✅ الحل جاهز

كل الكود شغال، بس محتاج تشغل SQL script واحد على Supabase

## 📋 الخطوات (5 دقائق)

### 1. افتح Supabase
- روح على: https://supabase.com/dashboard
- اختار المشروع بتاعك
- اضغط على **SQL Editor** من القائمة الجانبية

### 2. شغل الـ Script
- افتح ملف `FIX_PRODUCTION_SCHEMA.sql` من المشروع
- انسخ كل المحتوى
- الصقه في SQL Editor
- اضغط **Run** أو **F5**

### 3. استنى شوية
- الـ script هياخد من 10-30 ثانية
- هتشوف رسالة "Success" لما يخلص

### 4. اختبر الموقع
- افتح: https://tetiano.vercel.app
- سجل دخول
- افتح Console (F12)
- تأكد إن الأخطاء اختفت

- اعمل refresh للصفحة (Ctrl+F5)

### عايز تتأكد إن الـ Script اشتغل؟
شغل الـ query ده في SQL Editor:
```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
  AND table_name IN ('stores', 'store_memberships', 'shopify_connections');
```
